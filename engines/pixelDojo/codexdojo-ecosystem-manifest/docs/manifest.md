# codexDojo: Ecosystem Manifest

## codexDojo Purpose

`codexDojo` is a practical, continuous programming school operated by agents. It helps the learner
improve programming, architecture, code quality, testing, scalability, and professional AI usage by
building small projects that grow into robust systems.

The school is not a theory archive. Every cycle must produce useful artifacts: code, tests,
documentation, metrics, reviews, comparisons, memory updates, and a next challenge.

## Mechanics To Manifesto Mapping

| Mechanic | Manifesto mapping |
| --- | --- |
| Player movement through the dojo | The learner moves through active practice instead of reading passively. |
| NPC agents | Specialized AI agents guide architecture, tests, review, metrics, memory, and challenge selection. |
| Collectible artifacts | Useful outputs are the win condition, not decorative lore. |
| Hazards | Bugs, flaky tests, tech debt, scope creep, and legacy monoliths reduce engineering quality. |
| Cycle progression | Learning repeats as a production loop: build, verify, improve, remember, and unlock the next task. |
| HUD metrics | Quality, coverage, architecture, AI mastery, and memory are visible operational signals. |
| localStorage memory log | Completed cycles persist as auditable learner memory between sessions. |

## Why Every Cycle Produces Artifacts

Each cycle has a required artifact list. The player cannot complete a cycle by talking to agents or
standing in the dojo. Completion happens only after the required artifacts are collected.

Cycle 1 proves the first useful artifact exists by requiring:

- Code Scroll
- Test Shield
- Docs Map
- Memory Crystal

Cycle 2 improves the system by requiring:

- Review Seal
- Metrics Gem
- Comparison Mirror
- Next Challenge Key

This keeps the game aligned with the product rule: learning is complete only when useful evidence
exists.

## How Agents Operate The Learning Ecosystem

- Architect Agent keeps the system small and robust before scaling.
- Test Agent turns behavior into executable proof.
- Review Agent finds risks and quality gaps.
- Metrics Agent measures runtime, coverage, complexity, and AI reliance.
- Memory Agent preserves reusable learning instead of raw chat history.
- Challenge Agent turns the completed cycle into the next project.

Together, the agents operate the school as a loop. They do not replace the learner; they make the
learner's work visible, testable, reviewable, and ready for the next challenge.
